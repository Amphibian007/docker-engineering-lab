# Docker Hands-On

> A long-term, project-based knowledge base for learning Docker from fundamentals to production.

## Overview

This repository documents my journey of learning and applying Docker through real-world projects, experiments, architecture documentation, troubleshooting guides, and production concepts.

Rather than serving as a collection of isolated tutorials, this repository is organized as a structured knowledge base that grows over time. Each project builds practical experience, while the accompanying documentation captures the concepts and lessons learned along the way.

## Objectives

* Learn Docker through hands-on projects.
* Understand how containers are used in modern software development.
* Build production-oriented knowledge of containerized applications.
* Document concepts, architecture, and best practices.
* Maintain a professional reference for future projects and continuous learning.

## Projects

Hands-on projects are organized inside the `projects/` directory.

## Concepts

The `concepts/` directory contains notes and explanations covering Docker fundamentals, container lifecycle, networking, storage, image management, security, and production practices.

## Architecture

Architecture diagrams are maintained separately to visualize how Docker components interact in different environments.

## Troubleshooting

Quick-reference notes for commonly used Docker commands, Docker Compose, and Dockerfile instructions.

## Resources

Curated references to official documentation, books, blogs, articles, videos, and other learning materials.
